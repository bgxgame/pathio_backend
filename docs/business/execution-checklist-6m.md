# Pathio 6个月执行清单（PLG，UI冻结）

- 周期：24 周
- 原则：不改 UI 视觉；优先后端能力、增长闭环、数据可观测

## 阶段 1（第1-4周）：闭环可跑
- [ ] 上线 entitlement 表与双市场种子数据
- [ ] 上线 billing API：plans/subscription/checkout-session/webhook
- [ ] 上线事件 API 与 allowlist
- [ ] 前端接入 upgrade/share 埋点
- [ ] 联调 402 -> 升级 -> 支付成功 -> 解限

## 阶段 2（第5-8周）：转化提效
- [ ] 用真实支付网关替换 mock webhook
- [ ] checkout 失败/取消/退款分支打通
- [ ] 建立周报仪表盘（激活率、升级率、付费率）
- [ ] 校验中文/全球市场价格映射

## 阶段 3（第9-12周）：留存强化
- [ ] 激活“首图/首连线/首笔记/首分享”漏斗监控
- [ ] 组织级使用深度指标（周活跃节点、笔记更新率）
- [ ] 团队协作相关的留存实验（不改视觉）

## 阶段 4（第13-18周）：收入扩张
- [ ] Team 席位扩容策略（年付、阶梯）
- [ ] 企业线索收集（Enterprise 咨询入口与 CRM 同步）
- [ ] 关键客户场景梳理与案例页沉淀

## 阶段 5（第19-24周）：规模化准备
- [ ] 支付与订阅异常恢复策略
- [ ] 数据治理（事件命名、口径文档、审计）
- [ ] 下一周期路线：是否前置 Enterprise 交付

## 验收标准
- [ ] 402 升级闭环稳定
- [ ] 双市场价格与权益匹配
- [ ] 每周可见核心漏斗指标
